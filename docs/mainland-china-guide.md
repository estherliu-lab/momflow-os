# 中国大陆用户使用与部署方案

MomFlow 可以面向中国大陆用户使用。关键是把“网页/PWA 访问”和“AI 生成接口”分开设计。

## 1. PWA 本身

前端是静态 PWA，可以部署到大陆用户可访问的云服务或 CDN。建议：

- 不依赖 Google Fonts、海外 CDN 或海外统计脚本。
- 图片、图标和离线页都放在项目 `public/` 目录。
- 如果使用中国大陆服务器和正式域名，通常需要完成 ICP 备案。
- 如果涉及互动、账号、评论或公开发布能力，还要继续关注公安联网备案、内容安全与平台规则。

## 2. AI 供应商建议

不要把海外模型当成唯一接口。设置页已经预留这些大陆用户更容易访问的选项：

- DeepSeek
- 阿里云百炼 / 通义千问
- 智谱 GLM
- 火山方舟 / 豆包
- 百度千帆 / 文心
- 自定义 OpenAI-compatible API

MVP 阶段可以让用户自行填写 API Key。正式商业版本建议增加后端代理，不要让 API Key 暴露在浏览器里。

## 3. 推荐上线架构

```text
PWA 前端 -> 你的后端 /api/generate -> 大模型供应商 API
```

后端代理的好处：

- API Key 不暴露给浏览器。
- 可以做用户登录、额度、限流和账单。
- 可以增加内容安全审核。
- 可以根据地区自动选择可用模型。
- 可以统一记录错误、延迟和成本。

## 4. 内容是否能与时俱进

仅靠本地模板不会真正与时俱进。本地模板适合 MVP、演示和离线兜底。正式版本建议：

- 用模型输出结构化内容。
- 定期更新 prompt 模板，跟随平台风格变化。
- 增加热点/选题来源，例如搜索、RSS、自有选题库或人工维护的趋势库。
- 对生成结果做敏感词、广告法、平台规则检查。

## 5. 当前项目状态

当前版本已经支持：

- 本地模板兜底
- AI Provider 预设
- API Key / baseUrl / model 配置
- 中文/英文完整切换
- IndexedDB 本地保存
- PWA 安装、离线页、通知权限入口

下一步建议：

- 增加后端 AI 代理
- 增加内容安全审核
- 增加模型连通性测试按钮
- 增加热点选题源
- 增加云同步和账号系统
